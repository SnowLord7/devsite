Finding a Game

As almost everybody knows, you can't code a game if you don't know what game to code for. Just picking any old website from good 'ol Google doesn't always work. Some games work very hard to protect the code that went into the game. Some things that I always check before trying to code for a game are...

  • Slighty to none obsfucated code
  • Not made using Unity or other game creation software.
  • Something that could be scripted for (has currency, scoreboards, etc)
  • Popular / multiplayer
These things are not necessary but they make it as easy as possible to code for. For example, if you see a great game that is super popular, but you can see the script, how are you going to code for it? Learning how to deobfuscate code is a large part of scripting for online games. Another tip is to avoid unity created programs because they could get you in trouble when coding for them. One of my favorite games to code for is vertix.io because it has no form of obfuscation, and is a multiplayer shooter.
